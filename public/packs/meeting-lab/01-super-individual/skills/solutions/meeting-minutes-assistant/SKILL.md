---

# 什么时候触发
当用户已经有一段会议转写、飞书纪要、企微消息、钉钉会议记录或零散笔记，希望不要停在原始记录，而是先拿到一版结构化、可发送、可执行的会议纪要时触发。

# 目标
把一场会议压成：
- 背景摘要
- 核心结论
- 已决策事项
- 待办动作表
- 未决问题
- 发送版纪要正文
- 会后跟进检查单
- 下一句续跑指令

# 输入
- 会议主题
- 参会人 / 角色
- 原始记录（转写 / 笔记 / 聊天记录）
- 当前约束（哪些已定、哪些不能误判、输出格式偏好）

# 输出
至少输出下面 8 类内容：
1. 会议背景摘要
2. 核心结论
3. 已决策事项
4. 待办动作表
5. 未决问题清单
6. 发送版纪要正文
7. 会后跟进检查单
8. 下一句续跑指令

# 最小步骤
1. 先还原会议主题和背景
2. 区分讨论、结论、决定、待办和未决问题
3. 拉出动作项并尽量补 owner / deadline
4. 压成发送版纪要正文
5. 最后补跟进检查单和下一句续跑指令

# 强制规则
- 不要把讨论误写成已定结论
- 不要把待确认事项误写成已决策事项
- 如果原始记录里出现 owner / deadline，要优先显式抽出
- 如果信息不完整，要明确写“待确认”，不要擅自补定案
- 输出必须是能直接发送的，不只是摘要

# 常见坑
- 只给总结，不给动作项
- 已决策和未决混在一起
- 漏 owner、漏截止时间
- 语气太像会议分析，不像发送版纪要

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 背景
- 结论
- 已决策事项
- 待办动作表
- 未决问题
- 发送版纪要正文
- 检查单
- 下一句续跑指令
