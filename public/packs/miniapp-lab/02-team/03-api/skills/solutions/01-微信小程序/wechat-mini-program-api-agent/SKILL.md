# 什么时候触发
当团队协作版需要该角色处理其专属产物时触发。

# 输入
- 上一角色交付物
- 当前角色职责
- MVP 边界
- 当前约束

# 输出
- 当前角色负责的结构化产物

# 最小步骤
1. 读取上游交付物
2. 只处理自己职责边界
3. 产出当前角色交付物
4. 标出需要下游接收的内容

# 常见坑
- 越界做别的角色的事情
- 不给下游明确交接物
- 重新扩大发散需求

# 验证方式
检查输出是否只覆盖当前角色职责，并能交给下游。
