# 微信小程序产品拆解专家 示例输出

## 一句话场景
把“活动发现 -> 报名提交 -> 查看我的报名”压成一版可开工的小程序 MVP。

## MVP 边界
### 本期做
- 活动列表
- 活动详情
- 报名表单
- 报名成功页
- 我的报名
- 报名详情

### 本期不做
- 支付
- 优惠券
- 分销
- 消息推送
- 后台运营端

## 页面清单与职责
- 活动列表：展示活动卡片、状态、报名入口
- 活动详情：展示活动说明、时间地点、剩余名额、报名按钮
- 报名表单：收集姓名、手机号、公司、备注并提交
- 报名成功页：反馈提交成功并引导去“我的报名”
- 我的报名：展示用户已报名活动列表
- 报名详情：展示单条报名信息、活动信息、当前状态

## 关键操作流
1. 用户进入活动列表
2. 点击活动卡片进入详情
3. 点击报名进入表单
4. 提交成功后进入成功页
5. 在“我的报名”查看记录和详情

## 关键状态
- 活动状态：报名中 / 已结束 / 已报满
- 报名记录状态：已提交

## 交接物
### 给 builder
- 页面清单与页面职责
- 关键操作流
- 关键状态与页面跳转关系

### 给 api
- 实体：activity / signup / user
- 报名表单字段：name、phone、company、note
- 首批接口优先级：活动列表、活动详情、报名提交、我的报名列表、报名详情

### 给 qa
- 首轮验收范围：列表可进入详情、报名可提交成功、我的报名可看到新记录
