# 什么时候触发
当用户已经有一个明确主题、产品点、案例、观点或素材，希望不要停在抽象 brainstorming，而是先拿到一版标题、开头、正文、封面短句、评论引导和发布检查单时触发。

# 目标
把一句话主题压成：
- 选题角度
- 标题候选
- 开头钩子
- 正文底稿
- 封面短句 / 配图提示
- 评论区引导句
- 发布检查单
- 下一句续跑指令

# 输入
- 主题 / 产品 / 观点
- 面向谁
- 这篇想让读者得到什么
- 当前约束（不能太硬广、不要太术语化、控制字数等）

# 输出
至少输出下面 8 类内容：
1. 一句话角度
2. 3～5 个标题候选
3. 开头钩子（前 3 句）
4. 一版正文底稿
5. 封面短句
6. 配图 / 截图建议
7. 评论区引导句
8. 发布检查单 + 下一句续跑指令

# 最小步骤
1. 先收敛这一篇真正要打的角度
2. 先给多个标题方向
3. 先写开头钩子
4. 再出正文底稿
5. 再补封面短句 / 配图建议 / 评论区引导
6. 最后给发布检查单和下一句续跑指令

# 强制规则
- 不要只给平台建议，要给可直接改写的内容结果
- 不要只给 1 个标题，至少给 3 个
- 语气优先口语化、真实、像创作者在说话
- 如果用户主题偏硬，先帮他翻成平台语言，而不是照抄专业术语
- 如果用户素材不足，也先给一版可继续补素材的内容骨架

# 常见坑
- 只给选题，不给正文
- 只给正文，不给标题和开头
- 语气太像品牌文案，不像真实创作者
- 只讲方法，不给发前检查单

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 角度
- 标题
- 开头
- 正文
- 封面短句 / 配图建议
- 评论区引导
- 检查单
- 下一句续跑指令
