---
name: ppt-assistant
description: 单 Agent 帮用户把一个主题直接压成“可继续做成 PPT 的整套稿”。
---

# 什么时候触发
当用户已经有一个明确主题、汇报任务、路演题目或复盘方向，希望不要停在抽象 brainstorming，而是先拿到一版页序、逐页要点、图表建议和讲稿备注时触发。

# 目标
把一句话主题压成：
- 叙事切口
- 页序结构
- 每页标题
- 每页要点
- 图表 / 配图建议
- 开场 / 结尾建议
- 演讲备注
- 做版前检查单
- 下一句续跑指令

# 输入
- 主题 / 场景 / 汇报对象
- 想让听众得到什么
- 当前已有素材
- 当前约束（时间、页数、风格、不能展开什么）

# 输出
至少输出下面 9 类内容：
1. 一句话叙事切口
2. 页序结构
3. 每页标题
4. 每页核心要点
5. 图表 / 配图建议
6. 开场页建议
7. 结尾页 / CTA / Q&A 建议
8. 做版前检查单
9. 下一句续跑指令

# 最小步骤
1. 先收敛这份 PPT 真正要打的主线
2. 先排页序
3. 再给每页标题和要点
4. 再补图表 / 配图 / 备注建议
5. 最后给做版前检查单和下一句续跑指令

# 强制规则
- 不要只给“建议逻辑清楚”，要给整套页序与逐页结果
- 不要只给 3 页概念，要给能继续落版的整套结构
- 如果用户主题偏散，先帮他收成一条叙事主线
- 图表建议必须尽量贴近每页内容，而不是泛泛地说“可放数据图”
- 如果素材不足，也先给一版可继续补素材的逐页骨架

# 常见坑
- 只给主题，不给页序
- 只给页序，不给逐页要点
- 没有图表 / 配图建议
- 只讲设计感，不给做版前检查单

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 主线
- 页序
- 每页标题
- 每页要点
- 图表 / 配图建议
- 检查单
- 下一句续跑指令
