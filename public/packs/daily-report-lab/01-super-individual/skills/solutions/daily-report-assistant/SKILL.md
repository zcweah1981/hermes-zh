---
name: daily-report-assistant
description: 单 Agent 帮用户把一天进展直接压成“可发送、可跟进的项目日报”。
---

# 什么时候触发
当用户已经有一天的零散工作记录、待办、群消息、Git 提交、会议同步内容，希望不要停在原始记录，而是先拿到一版结构化、可发送、可跟进的项目日报时触发。

# 目标
把一天进展压成：
- 今日摘要
- 完成事项
- 进行中事项
- 阻塞 / 风险
- 明日计划
- 发送版日报正文
- 跟进检查单
- 下一句续跑指令

# 输入
- 今日目标
- 原始记录（待办 / 群消息 / 笔记 / Git 提交 / 会后内容）
- 当前约束（哪些已完成、哪些不能误判、输出格式偏好）

# 输出
至少输出下面 8 类内容：
1. 今日摘要
2. 完成事项
3. 进行中事项
4. 阻塞 / 风险
5. 明日计划
6. 发送版日报正文
7. 跟进检查单
8. 下一句续跑指令

# 最小步骤
1. 先还原今天围绕什么目标在推进
2. 区分完成、进行中、阻塞和明日计划
3. 抽出需要协助或重点关注的事项
4. 压成发送版日报正文
5. 最后补跟进检查单和下一句续跑指令

# 强制规则
- 不要把推进中误写成已完成
- 不要把待确认风险写成已经解决
- 如果原始记录里出现 owner / deadline / 依赖项，要优先显式抽出
- 如果信息不完整，要明确写“待确认”或“进行中”，不要擅自补完成状态
- 输出必须是能直接发送的，不只是摘要

# 常见坑
- 只给总结，不给完成项和阻塞项
- 完成、进行中、阻塞混在一起
- 漏风险、漏需要协助的地方
- 语气太像复盘，不像发送版日报

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 今日摘要
- 完成事项
- 进行中事项
- 阻塞 / 风险
- 明日计划
- 发送版日报正文
- 检查单
- 下一句续跑指令
