# 什么时候触发
当用户已经有一个明确主题、案例、观点、产品更新或行业判断，希望不要停在抽象 brainstorming，而是先拿到一版标题、大纲、导语、正文、结尾收口和发布检查单时触发。

# 目标
把一句话主题压成：
- 文章切口
- 标题候选
- 文章大纲
- 导语
- 正文初稿
- 小标题 / 配图建议
- 结尾 CTA
- 发布检查单
- 下一句续跑指令

# 输入
- 主题 / 案例 / 观点
- 面向谁
- 这篇文章想让读者得到什么
- 当前约束（不能太像广告、不要太口语、控制篇幅等）

# 输出
至少输出下面 9 类内容：
1. 一句话切口
2. 3～5 个标题候选
3. 文章大纲
4. 文首导语
5. 一版正文初稿
6. 小标题建议
7. 配图 / 引用建议
8. 结尾 CTA
9. 发布检查单 + 下一句续跑指令

# 最小步骤
1. 先收敛这篇文章真正要打的切口
2. 先给多个标题方向
3. 先拉出文章大纲
4. 再写导语和正文初稿
5. 再补小标题 / 配图 / 结尾 CTA
6. 最后给发布检查单和下一句续跑指令

# 强制规则
- 不要只给写作建议，要给可继续润色的文章结果
- 不要只给 1 个标题，至少给 3 个
- 语气优先清楚、可信、像公众号作者在写，而不是营销口号
- 如果用户主题偏硬，先帮他翻成文章表达，而不是照抄内部术语
- 如果用户素材不足，也先给一版可继续补案例的文章骨架

# 常见坑
- 只给选题，不给大纲和正文
- 只给正文，不给导语和结尾收口
- 语气太像广告文，不像公众号文章
- 只讲方法，不给发布检查单

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 切口
- 标题
- 大纲
- 导语
- 正文
- 小标题 / 配图建议
- 结尾 CTA
- 检查单
- 下一句续跑指令
