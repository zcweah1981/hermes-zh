---

# 什么时候触发
当用户已经有一篇现成内容（公众号文章、飞书文档、演讲逐字稿等），希望不要只发一个平台，而是直接拿到多平台可发布版本时触发。

# 目标
把原始内容改写成：
- 小红书版本（标题带 emoji、正文口语化、末尾带话题标签）
- 公众号版本（深度完整、小标题清晰、引导关注）
- X/Twitter 版本（精炼犀利、拆成 1-3 条推文）
- 每个版本都是可以直接复制发布的完整稿件
- 下一句续跑指令

# 输入
- 原始内容全文
- 目标平台列表
- 品牌调性约束
- 各平台特殊要求

# 输出
至少输出下面内容：
1. 原文核心信息提炼
2. 小红书完整版本
3. 公众号适配版本
4. X/Twitter 推文版本
5. 各平台发布检查项
6. 下一句续跑指令

# 最小步骤
1. 先提炼原文核心信息和关键数据点
2. 改写小红书版本：标题加钩子、正文口语化、加 emoji 和话题
3. 改写公众号版本：补导语、调结构、加收口
4. 改写 X/Twitter 版本：提炼核心做主推文、补 1-2 条展开推文
5. 补各平台发布前检查项
6. 最后补下一句续跑指令

# 强制规则
- 每个版本都必须保留原文核心信息，不丢关键数据
- 小红书标题必须有钩子（emoji + 悬念/痛点/数字）
- X 推文每条不超过 280 字符
- 不要把同一个版本直接复制到另一个平台
- 输出必须是可以直接复制发布的，不只是改写建议

# 常见坑
- 只出一个平台的改写，漏了其他平台
- 小红书版本太正式，缺少口语感
- X 推文太长，超出 280 字符限制
- 公众号版本丢失原文深度
- 各版本之间信息不一致

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 原文核心信息提炼
- 小红书完整版本（标题 + 正文 + 话题标签）
- 公众号适配版本
- X/Twitter 推文版本
- 各平台发布检查项
- 下一句续跑指令
