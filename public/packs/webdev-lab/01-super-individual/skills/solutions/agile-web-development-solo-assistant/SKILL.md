name: agile-web-development-solo-assistant
description: 单 Agent 帮用户把 Web 工具需求直接压成“可开工页面结构 + 接口草案 + 代码骨架”。
---

# 什么时候触发
当用户已经有一个明确的 Web 工具想法，希望不要停在需求说明，而是先拿到一版可开工的页面、数据、接口和文件骨架时触发。

# 目标
把一句话需求压成：
- 页面与路由清单
- 数据与接口草案
- Web 项目骨架建议
- 首批代码文件建议
- 开发顺序
- 测试检查单

# 输入
- 业务目标
- 目标用户
- 最小闭环
- 当前约束（不做什么）

# 输出
至少输出下面 7 类内容：
1. 页面清单
2. 功能清单
3. 数据结构建议
4. 接口建议
5. Web 目录/代码骨架建议
6. 开发顺序
7. 测试检查单

# 最小步骤
1. 收敛一句话场景
2. 压出 MVP 边界
3. 输出页面/功能
4. 输出数据/接口
5. 输出项目骨架与首批代码文件建议
6. 输出开发顺序
7. 输出测试检查单

# 常见坑
- 只给抽象建议，不往代码骨架走
- 范围过大
- 约束条件没写
- 把 UI 设计稿和实现包混成一个请求

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 页面
- 数据
- 接口
- 代码骨架
- 顺序
- 测试
