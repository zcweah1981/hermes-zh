name: material-summary-assistant
description: 单 Agent 帮用户把一批资料直接压成“可发送、可判断、可继续补的结构化总结”。
---

# 什么时候触发
当用户已经有一批零散资料、会议记录、网页内容、用户反馈或笔记，希望不要停在原始材料，而是先拿到一版结构化、可发送、可继续决策的总结时触发。

# 目标
把一批资料压成：
- 背景摘要
- 核心结论
- 关键信息保留项
- 待确认项
- 建议动作
- 发送版总结正文
- 补材料检查单
- 下一句续跑指令

# 输入
- 资料主题
- 原始材料（网页 / 记录 / 群消息 / 笔记）
- 当前约束（哪些已确定、哪些不能误判、输出格式偏好）

# 输出
至少输出下面 8 类内容：
1. 背景摘要
2. 核心结论
3. 关键信息保留项
4. 待确认项
5. 建议动作
6. 发送版总结正文
7. 补材料检查单
8. 下一句续跑指令

# 最小步骤
1. 先还原这批资料围绕什么问题
2. 区分背景、结论、保留项和待确认项
3. 抽出值得继续推进的建议动作
4. 压成发送版总结正文
5. 最后补检查单和下一句续跑指令

# 强制规则
- 不要把推测写成已确定结论
- 不要把背景信息误写成核心结论
- 如果原始资料里出现明确建议动作或待确认对象，要优先显式抽出
- 如果信息不完整，要明确写“待确认”或“需补材料”，不要擅自定案
- 输出必须是能直接发送的，不只是摘要

# 常见坑
- 只给概述，不给核心结论
- 结论、保留项、待确认项混在一起
- 漏建议动作、漏补材料提醒
- 语气太像研究随笔，不像发送版总结

# 验证方式
用 sample-input 跑一次，检查输出是否同时包含：
- 背景摘要
- 核心结论
- 保留项
- 待确认项
- 建议动作
- 发送版总结正文
- 检查单
- 下一句续跑指令
