# sample-input

资料主题：
整理一批关于“AI Agent 在小团队落地方式”的散资料，用于给老板做一次快速同步。

原始材料：
- 一篇文章提到：不要一上来追求全自动，要先选高频、低风险、可校验的流程
- 一段会议记录提到：团队现在最适合先试会议纪要、日报、FAQ 初稿这类结构化任务
- 一条飞书消息提到：当前最大问题不是工具不够，而是大家对输入输出口径不统一
- 一段用户反馈里提到：如果没有 owner、deadline、检查点，Agent 做再多也容易失控
- 一条企微消息提到：老板更关心这东西能不能先节省大家的重复整理时间，而不是技术有多酷

约束：
- 不要写成长篇研究报告
- 重点是给老板一版 3 分钟能看懂的总结
- 哪些还不能定论，请明确写出来

请先把这批资料压成一版可发送、可判断的结构化总结，输出：
1. 背景摘要
2. 核心结论
3. 关键信息保留项
4. 待确认项
5. 建议动作
6. 发送版总结正文
7. 补材料检查单
8. 下一句续跑指令
