# sample-output

## 背景摘要
这批资料主要围绕“AI Agent 在小团队怎么先落第一步”展开，核心不是技术细节，而是如何先选对场景、统一口径、建立最小可控流程。

## 核心结论
- 小团队不该一开始就追求全自动
- 更适合先从高频、低风险、可校验的结构化任务切入
- 当前最大的实际难点不是工具不足，而是输入输出口径不统一

## 关键信息保留项
- 会议纪要、日报、FAQ 初稿是当前最适合试点的任务
- 如果没有 owner、deadline、检查点，流程很容易失控
- 管理者更关心节省重复整理时间，而不是技术炫酷度

## 待确认项
- 当前团队内部是否已经有统一模板和验收口径
- 第一批试点任务中，谁来做 owner、谁来做校验

## 建议动作
- 先选一个高频结构化任务做试点
- 先统一输入输出模板
- 先把 owner、deadline、检查点写死再上 Agent
